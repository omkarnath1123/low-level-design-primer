# Mock Interview Prompts

Use these prompts for self-practice or peer mock interviews.

## Prompt 1: parking lot
Design a parking lot system.
Focus on:
- spot assignment
- vehicle types
- entry and exit
- pricing extension points
- concurrency around occupancy

## Prompt 2: LinkedIn
Design a professional networking platform like LinkedIn.
Focus on:
- profiles
- connections
- messaging
- feed or notification primitives
- search boundaries

## Prompt 3: splitwise
Design a bill-sharing system like Splitwise.
Focus on:
- balance tracking
- expense splitting strategies
- settlements
- invariants around amounts and debt

## Prompt 4: pub-sub
Design a pub-sub or message distribution system.
Focus on:
- publishers and subscribers
- topic routing
- retries
- ordering assumptions
- extensibility of delivery channels

## Prompt 5: movie ticket booking
Design a movie ticket booking system.
Focus on:
- show discovery
- seat selection
- reservation vs confirmation
- double-booking prevention
- payment integration boundary

## Prompt 6: ride sharing
Design a ride-sharing service.
Focus on:
- rider and driver states
- matching strategy
- trip lifecycle
- cancellation and retry behavior
- concurrent trip assignment risk

## Prompt 7: LRU cache
Design an LRU cache.
Focus on:
- API
- time complexity
- data structures
- thread-safety discussion

## Prompt 8: digital wallet
Design a digital wallet service.
Focus on:
- wallet balance
- money movement
- idempotency
- transaction ledger
- notifications

## Prompt 9: online shopping
Design an online shopping system.
Focus on:
- catalog
- cart
- inventory
- checkout lifecycle
- order state transitions

## Prompt 10: hotel management
Design a hotel management system.
Focus on:
- room inventory
- reservation lifecycle
- pricing
- booking conflicts
- extension for services or add-ons

## Follow-up question pack
Ask these after every design:
- What are the key invariants?
- What is the biggest concurrency risk?
- What would you persist?
- What would you cache?
- What would change in production?
- Where is the main extension point?
- Which part did you intentionally simplify?

## Scoring rubric
Rate each answer from 1 to 5 on:
- clarity
- object modeling
- correctness
- concurrency thinking
- tradeoff explanation
- extensibility
