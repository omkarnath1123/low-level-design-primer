# Study roadmap

This directory organizes the repository into guided preparation tracks for
low-level design and staff-level system design interviews.

Use these files to choose what to study next instead of randomly bouncing
between unrelated problems.

## Files in this directory
- `00-overview.md`: how to use the roadmap
- `01-by-difficulty.md`: easy, medium, and hard progression
- `02-by-company-style.md`: likely company-style buckets
- `03-by-theme.md`: domain and systems themes
- `04-8-week-roadmap.md`: suggested study plan
- `05-top-25-problems.md`: condensed must-practice shortlist
- `06-mock-interview-prompts.md`: ready-to-use mock prompts and rubric
- `07-revision-tracker.md`: manual revision and mock tracking sheet

## Recommended reading order
1. Read `00-overview.md`.
2. Start with `01-by-difficulty.md`.
3. Use `05-top-25-problems.md` to prioritize your practice.
4. Use `03-by-theme.md` to fill weak spots.
5. Use `02-by-company-style.md` for targeted interview prep.
6. Follow `04-8-week-roadmap.md` for structured practice.
7. Use `06-mock-interview-prompts.md` and `07-revision-tracker.md` for
   execution and review.

## How to use the roadmap well
- Pick one track at a time.
- Solve from prompts before reading implementations.
- Review tradeoffs, not just class names.
- Revisit hard problems after practicing easier ones.

This adds guidance, not a new architecture. The source material still lives in
other top-level repository directories.
