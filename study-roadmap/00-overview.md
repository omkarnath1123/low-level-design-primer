# Roadmap Overview

This repository now contains multiple kinds of study material:
- fundamentals
- design patterns
- problem statements
- class diagrams
- reference implementations
- cloned external solutions
- video walkthroughs

## How to use the repo well

### Step 1: Build design vocabulary
Start with:
- `fundamentals/oop-java/`
- `Design-Patterns.md`
- `Distributed-Systems-Pattern.md`

### Step 2: Solve from prompts first
Use `problem-statements/` without opening implementations immediately.
For each prompt:
1. rewrite requirements
2. identify entities and workflows
3. sketch classes and relationships
4. identify invariants and concurrency risks
5. state assumptions and non-goals

### Step 3: Compare solutions
Then compare against:
- `reference-implementations/java/`
- `solutions/`
- `assets/class-diagrams/`

### Step 4: Practice staff-level communication
For each design, answer:
- why these abstractions?
- what are the extension points?
- what is intentionally simplified?
- what breaks first at scale?
- what would I change in production versus interview scope?

## Important rule
Do not memorize class diagrams blindly.
Use them to sharpen judgment, not replace thinking.
